# Session Handoff

Read the portfolio board, `AGENTS.md`, product thesis, decision registry, data
manifest, submission ledger, current build, Git status, and recent commits.
Prepend a dated entry to `docs/session-handoff.md` with:

- verified deadlines/rules and P0/P1 boundary;
- selected or pending concept, rejected candidates, and current gate;
- accepted/pending datasets and license blockers;
- working interaction path, checks run, screenshots/deployment state;
- exact blockers, next artifact, and resume commands;
- branch, commit, and uncommitted-file ownership.

Never include credentials, private browser state, or vague chat-only conclusions.
