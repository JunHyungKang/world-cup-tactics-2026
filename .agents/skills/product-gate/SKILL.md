# Product Gate

Require one candidate card with:

- target user and one-sentence manager fantasy;
- decision the user makes, direct manipulation used, and visible consequence;
- real World Cup data needed and why it changes the decision;
- differentiation from a formation drag-and-drop board;
- 60-second demo path and first-screen comprehension test;
- smallest interactive prototype, owner files, test, time box, and stop signal;
- data/license risk, football-validity risk, and deployment complexity;
- expected judging value across originality, data credibility, interaction,
  visual clarity, reliability, and finishability.

Return `PASS`, `REVISE`, or `REJECT`. Reject concepts whose core loop is a
static dashboard, whose "AI" output cannot be judged without a private key, or
whose real-data claim cannot be licensed and reproduced. A synthetic prototype
may test UX only when visibly labeled and may not be promoted as football advice.

File the result in `docs/product-thesis.md` and `docs/decision-registry.md`.
