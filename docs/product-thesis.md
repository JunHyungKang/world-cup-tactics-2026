# Product Thesis

## Status

`pending-discovery` — no concept or football dataset has passed the product gate.
The current Touchline Lab screen is a UX shell, not the selected competition
thesis and not an evidence-backed match predictor.

## Required differentiator

The winning loop must go beyond arranging a formation. A user should inspect
opponent evidence, make one consequential manager choice, manipulate the tactic,
and understand the resulting tradeoff in under 60 seconds.

## Candidate-card template

For each candidate, record:

1. Manager fantasy and target user.
2. Decision, manipulation, consequence, and 60-second demo.
3. Real dataset and exact calculation needed.
4. Difference from a generic drag-and-drop board.
5. Data/license, football-validity, UX, and deployment risks.
6. Smallest prototype, time box, observable pass signal, and stop condition.
7. Scores from 1-5 for differentiation, data credibility, interaction, visual
   clarity, reliability, and finishability.

## Discovery queue

Rank at least three data-backed loops before selection, such as opponent matchup
planning, in-match adjustment, or role/space tradeoff. These are search directions,
not approved features. Reject any candidate that needs unavailable event data or
a private judge API key.
